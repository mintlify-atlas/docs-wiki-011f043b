Fineract uses a multi-project Gradle build declared in `settings.gradle`. As of the current codebase there are over 30 `include` directives covering core infrastructure, domain products, client libraries, test harnesses, and deployment artefacts. Understanding which module owns which domain prevents both duplicated code and unwanted circular dependencies.

## Custom Extension Pattern

The build supports a three-level namespace for company-specific extensions:

```
custom/
  <company>/
    <category>/
      <module>/   → included as :custom:<company>:<category>:<module>
```

This is dynamically discovered via `settings.gradle`:

```groovy
file("${rootDir}/custom").eachDir { companyDir ->
    if ('build' != companyDir.name && 'docker' != companyDir.name) {
        file("${rootDir}/custom/${companyDir.name}").eachDir { categoryDir ->
            file("${rootDir}/custom/${companyDir.name}/${categoryDir.name}").eachDir { moduleDir ->
                include ":custom:${companyDir.name}:${categoryDir.name}:${moduleDir.name}"
            }
        }
    }
}
```

Custom modules can declare a dependency on any published Fineract module and contribute Spring `@Configuration` classes that are picked up via auto-configuration.

## Build Configuration Highlights

**Develocity build scans** (`settings.gradle`):

```groovy
develocity {
    projectId = "fineract"
    server = "https://develocity.apache.org"
    buildScan {
        uploadInBackground = !isCI
        publishing.onlyIf { it.authenticated }
    }
}
```

Build scans are uploaded to the Apache Develocity instance. On CI (`JENKINS_URL` set), uploads are synchronous; locally they run in the background. IP addresses in scans are obfuscated to `0.0.0.0`.

**Local build cache** is enabled by default (`buildCache.local.enabled = true`), meaning incremental compilation results are cached across Gradle invocations on the developer machine.

<Note>
Gradle configuration in `buildSrc/` contains shared logic: the `fineractJavaProjects` and `fineractPublishProjects` extension functions that configure checkstyle, PMD, SpotBugs, and Maven publication settings consistently across all submodules.
</Note>
