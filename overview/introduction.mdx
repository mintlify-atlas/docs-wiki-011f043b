Apache Fineract is an open-source, open-core banking platform maintained by the Apache Software Foundation. It exposes a multi-tenant REST API covering the full lifecycle of loans, savings products, accounting, and batch close-of-business (COB) processing. Financial service providers — from microfinance institutions (MFIs) to fintech digital lenders — deploy Fineract as their core banking system, connecting it to mobile money, payment gateways, and reporting engines.

## What Fineract Provides

Fineract ships as a self-contained Spring Boot application (`ServerApplication`) that can be run directly in an IDE or packaged as a WAR. Its primary capabilities are:

- **Loan origination and lifecycle** — application, approval, disbursement, repayment, rescheduling, write-off, and closure across conventional, Islamic, and progressive loan products.
- **Savings and deposit accounts** — voluntary savings, recurring deposits, fixed deposits, and share accounts.
- **General ledger and accounting** — double-entry accounting, journal entry automation, periodic accruals, and journal entry aggregation batch jobs.
- **Close-of-Business (COB) batch** — daily Spring Batch jobs that accrue charges, apply overdue penalties, update delinquency classifications, and emit business events per loan.
- **External event streaming** — domain events serialized to Apache Avro and published to Kafka or ActiveMQ for downstream consumers.
- **Multi-tenancy** — each HTTP request carries a `Fineract-Platform-TenantId` header; the platform resolves a fully isolated per-tenant datasource at runtime.

## Who Uses It and How It Is Deployed

Fineract is deployed by MFIs, rural banks, neobanks, and NGO lenders in Africa, Asia, and Latin America. It is typically run on-premise or in a cloud container environment (Docker/Kubernetes). A Docker Compose reference stack is included at `custom/docker/`. Deployments commonly pair Fineract with:

- A PostgreSQL or MariaDB/MySQL database (one schema per tenant, plus a shared `fineract_tenants` metadata database).
- An Apache Kafka cluster for external business events.
- An external identity provider (Keycloak) when OAuth2 mode is enabled.

## Brief History

The codebase originated as Mifos (Microfinance Open Source), donated to the Apache Software Foundation in 2012. It graduated from the Apache Incubator in 2016 and has been maintained as a top-level Apache project since. The modern architecture replaced a legacy WAR/Tomcat-only deployment with an embedded Spring Boot `SpringApplicationBuilder`, enabling IDE-level debugging without a separate application server.

## Technology Stack

| Layer | Technology |
|---|---|
| Language | Java 17+ |
| Application framework | Spring Boot 3.x (embedded Tomcat) |
| Persistence | JPA / Hibernate + Spring Data JPA |
| Batch processing | Spring Batch (partitioned, multi-threaded) |
| Schema migration | Liquibase |
| Databases | PostgreSQL 14+, MariaDB 10.6+, MySQL 8+ |
| Serialization | Apache Avro (external events), Jackson (REST) |
| Messaging | Apache Kafka, ActiveMQ (JMS) |
| Security | Spring Security (Basic Auth + OAuth2 JWT) |
| Build system | Gradle 8 with Develocity build scans |
| License | Apache License 2.0 |
